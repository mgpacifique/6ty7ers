# Database Schema — Data Dictionary

PostgreSQL schema for the queue system. The source of truth is
`backend/app/models.py` and the Alembic migration `79dfbd94357c_initial_schema`.
This document describes every table, column type and key, and shows the sample
rows loaded by `backend/app/seed_sample_data.py`.

## Entity overview

```
patients 1 ──── * queue_sessions * ──── 1 staff   (triaged_by / consulted_by)
                      │
                      └── events recorded in system_logs
```

## Table: patients

Stores private patient identity. Never shown on public screens — public
displays only ever use `queue_sessions.public_token`.

| Column | Type | Key / Constraint | Notes |
|---|---|---|---|
| id | UUID | PK | generated by the app |
| full_name | VARCHAR | NOT NULL | private (PII) |
| phone_number | VARCHAR | NOT NULL | for notifications |
| created_at | TIMESTAMPTZ | | set on insert (UTC) |

## Table: staff

| Column | Type | Key / Constraint | Notes |
|---|---|---|---|
| id | UUID | PK | |
| username | VARCHAR | UNIQUE, NOT NULL, indexed | login handle |
| password_hash | VARCHAR | NOT NULL | bcrypt hash, never plain text |
| role | VARCHAR | NOT NULL | `Admin`, `Nurse` or `Doctor` |
| created_at | TIMESTAMPTZ | | |

## Table: queue_sessions

One row per patient visit. This is the queue itself.

| Column | Type | Key / Constraint | Notes |
|---|---|---|---|
| id | UUID | PK | |
| patient_id | UUID | FK → patients.id, NOT NULL | |
| public_token | VARCHAR | UNIQUE, NOT NULL, indexed | `EM-###` urgent, `FT-###` routine |
| track_type | VARCHAR | nullable | `Urgent` or `Routine`, set at triage |
| priority_score | INTEGER | default 0 | calculated by the Smart Logic Engine |
| status | VARCHAR | | `Registered` → `Triaged` → `Waiting` → `Called` → `Completed` |
| t1_check_in | TIMESTAMPTZ | | T1: arrival time |
| t2_called | TIMESTAMPTZ | nullable | T2: called to room |
| t3_completed | TIMESTAMPTZ | nullable | T3: session end |
| triaged_by_staff_id | UUID | FK → staff.id, nullable | nurse who triaged |
| consulted_by_staff_id | UUID | FK → staff.id, nullable | doctor who consulted |

KPI: **true wait time = t2_called − t1_check_in**, straight from the columns.
